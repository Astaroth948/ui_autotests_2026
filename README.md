# UI Автотесты (Pytest + Playwright + Allure)

Данный readme файл пока в работе как и недостающие docstring.

## Описание
Это тестовый проект по автоматизации UI тестов, предназначенный для демонстрации моих навыков в работе с необходимыми инструментами.

В качестве тестового приложения используется приложение с [курса](https://stepik.org/course/234842/promo) по [адресу](https://nikita-filonov.github.io/qa-automation-engineer-ui-course/#/auth/login).

Данный код не является копией кода с курса, оттуда используется только само приложение и несколько удачных на мой взгляд решений, все остальное было полностью переработано с учетом моего опыта и навыков, плюс реализовано несколько моих собственных задумок, которых не найти ни в одном курсе.

## Особенности проекта
- Большое количество тестов (81).
- Архитектура с использованием паттернов Page Object, Page Component и Page Factory.
- Общие фикстуры, вынесенные в плагины.
- Расширенная работа с хуками pytest.
- Тестовые файлы можно запускать по разной логике через проставленные метки pytset. Также были добавлены отдельные метки для демонстрации падающих тестов с разными scope.
- Обработка падающих тестов:
    - В качестве демонстрации падающих тестов были созданы отдельные тесты.
    - Аттачи разбиты по категориям (setup, call и teardown). Также если тест падает на фикстуре и в теле теста, то аттачи будут и там и там.
    - Проброс аттачей падающих фикстур во все тесты, где они есть. Обычно если тест падает на фикстуре со scope class, module или session, то аттачи будут только на одном тесте, в данном фреймворке эта проблема исправлена.
- Кастомизация отчета:
    - Кастомные категории отчета. Я старался по максимуму использовать все категории, доступные в allure. Также есть возможность обработки тестов с заведенными багами и вынесение их в отдельные редкоиспользуемые категории для читабельности отчета (unknown и broken).
    - Кастомные шаги в отчете. Каждый шаг не просто описывает действие или проверку, но и указывает на какой сущности страницы это происходит с полным путем.
    - Отсутствие allure тегов. Вместо этого в отчете отображаются метки pytest, переведенные в формат UPPER_SNAKE_CASE.
- Использование .env файла для всех необходимых настроек, включая кастомизацию отчетов
- Присутствует настройка для очистки директории с allure отчетами перед каждым запуском прогона, используется для локальной отладки тестов.

## Запуск фреймворка
Установка библиотек:

```
pip install -r requirements.txt
```
Фреймворк настроен тестовым .env.local файлом и сразу готов к запуску.

Запуск основных тестов:
```
python -m pytest -m "regression or smoke or failed_function or failed_class or fixed_function"
```

Тесты с метками ```failed_session``` и ```failed_module``` необходимо запускать по отдельности, т.к. они открывают еще один экзепляр браузера и могут сбить работу основных тестов. Сами по себе они ничего не проверяют, просто показывают логику пробрасывания аттачей в отчете.
